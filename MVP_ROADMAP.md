# אוספים עובדות - מפת המשך ל-MVP

## מצב נוכחי

- אפליקציית Angular משחקית עם חבילות, אלבום, פתיחת חבילה, קלפים ובוחן.
- PWA מוגדר בפרויקט Angular: manifest, icons, service worker ו-cache production.
- בקאנד Express פרוס ב-Railway תחת `fact-collectors-backend`.
- סכימת Postgres/Supabase מוכנה תחת `backend/supabase/schema.sql`.
- הקטלוג כולל 13 חבילות, 130 קלפים ו-10 שאלות לכל קלף.

## מה חסר לפני MVP אמיתי

- להפעיל את `backend/supabase/schema.sql` ב-Supabase.
- להגדיר ב-Railway את `SUPABASE_URL` ואת `SUPABASE_SERVICE_ROLE_KEY`.
- לחבר את Angular ל-API במקום לשמור state רק ב-localStorage.
- להוסיף זיהוי משתמש בסיסי כדי ששחקן יוכל להמשיך מהמכשיר שלו.
- להוסיף מסכי מצב: טעינה, שגיאת רשת, אין מטבעות, ניסיון מחר.

## מפת פיתוח מומלצת

### שלב 1 - Persistence אמיתי

- יצירת שחקן מול הבקאנד.
- שמירת מטבעות, קלפים שנחשפו, קלפים שנאספו, כפילויות וניסיונות בוחן ב-Postgres.
- סנכרון state אחרי פתיחת חבילה ואחרי סיום בוחן.

### שלב 2 - MVP משחקי

- חנות ראשונית לקניית חבילות/מטבעות דמו.
- Daily reward כדי לאפשר המשך משחק בלי תשלום.
- אנימציות reward: קלף חדש, כפילות, מעבר בוחן, כישלון וניסיון מחר.
- מסך פרופיל שחקן: רמה, מטבעות, אחוז השלמת אלבום.

### שלב 3 - איכות מוצר

- Lighthouse/PWA audit.
- בדיקות רספונסיביות למובייל קטן, טאבלט ודסקטופ.
- בדיקות API בסיסיות לפתיחת חבילה ובוחן.
- מנגנון seed/migration לקטלוג קלפים ושאלות.

### שלב 4 - לקראת מוצר ציבורי

- Supabase Auth או login קליל להורים/ילדים.
- Admin קטן לניהול חבילות, קלפים ושאלות.
- מערכת עונות ותוכן מתחלף.
- אנליטיקות: פתיחות חבילה, קלפים פופולריים, אחוזי הצלחה בבוחנים.

## סדר עדיפויות הקרוב

1. לחבר Supabase אמיתי ולהחליף את fallback הזיכרון.
2. לחבר את Angular לבקאנד.
3. לסיים PWA audit ו-install flow.
4. לייצב את החנות והכלכלה הראשונית.
5. להוסיף onboarding קצר לשחקן חדש.
